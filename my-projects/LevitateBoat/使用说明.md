# Levitate Boat 使用说明

本文档面向最终用户，说明如何安装、配置与合理使用本插件，并理解其设计理念与「持续学习」机制。

---

## 一、安装与加载

1. **构建项目**（若你从源码安装）  
   ```bash
   npm install
   npm run build
   ```
2. 打开 Chrome，进入 `chrome://extensions/`。  
3. 开启右上角「**开发者模式**」。  
4. 点击「**加载已解压的扩展程序**」，选择项目下的 **`dist`** 文件夹。  
5. 安装完成后，浏览器工具栏会出现扩展图标；点击图标即可打开侧栏主界面。

---

## 二、基本配置（设置页）

点击主界面右上角**齿轮图标**进入设置页。

### 2.1 API 配置

- **API 类型**：Ollama（本地）、OpenAI、Claude、自定义（如 DeepSeek）。  
- **API 地址**：例如 DeepSeek 填 `https://api.deepseek.com/v1`，Ollama 填 `http://localhost:11434`。  
- **API Key**：使用 OpenAI/Claude/自定义时必填；仅保存在本地，不会上传到本插件服务器。  
- **Chat 模型**：如 `deepseek-chat`、`gpt-4` 等。  
- **Embedding 模型**（可选）：用于记忆语义检索；不填则使用简单关键词检索。

配置完成后点击「**保存配置**」。

### 2.2 用户画像（回复风格、自定义说明、自定义标签）

- **回复风格**：简洁 / 适中 / 详细，影响助手整体回答长度与详略。  
- **自定义说明**：自由填写你的偏好或禁忌（如「少用术语」「不要推荐购物」「偏好分点回答」），助手会在回答与提取记忆时参考。  
- **自定义标签**：  
  - 在输入框中输入标签名称，按**回车**或点击「**添加**」即可加入列表。  
  - 列表中每个标签旁有「×」可删除。  
  - 这些标签会在**自动提取记忆**时作为「建议优先使用的标签」传给模型，使记忆的分类更符合你的习惯。  
  - 保存后需点击「**保存画像**」才会生效。

画像区还会展示**近期常问**、**常用标签**、**最后更新**，以及可折叠的**标签云**，便于你了解自己的使用习惯。

**自动添加**：  
- **近期常问**：每次你在侧栏发送提问并得到回复后，该问题会**自动**计入「近期常问」统计（按频率排序）。  
- **常用标签**：每次对话结束后插件**自动提取记忆**并打上标签时，这些标签会**自动**计入「常用标签」与标签云。  
无需手动操作，使用越多，统计越能反映你的习惯。

**手动删除与整理**（设置页用户画像区域）：  
- **清空自动统计**：点击「清空自动统计」可清除「近期常问」和「常用标签」的全部自动记录，**不会**影响你已设置的回复风格、自定义说明和自定义标签。  
- **常用标签同步到自定义**：点击后会把「常用标签」里你尚未加入「自定义标签」的标签，批量加入自定义标签，方便后续提取记忆时优先使用。  
- **重置用户画像**：点击「重置用户画像」并确认后，将恢复默认回复风格、清空自定义说明与自定义标签，以及所有自动统计；相当于重新开始。

### 2.3 导出与导入

- **按对话导出记忆**（推荐）：  
  - 在主界面**左侧对话列表**中，**右键**某一对话，选择「**导出本对话记忆**」，即可仅将该对话下的记忆及所用标签导出为一个 JSON 文件（文件名含对话标题与时间戳）。每个对话的记忆是分开的，按需导出即可。  
- **导出全部记忆**（设置页）：  
  - 在设置页点击「**导出记忆**」会**一次性**将当前**全部记忆**和**全部标签**导出为**一个** JSON 文件（文件名形如 `levitate-boat-时间戳.json`），用于整体备份或迁移。  
- **导入记忆**：选择之前导出的 JSON 文件，可恢复记忆与标签（不会清空现有数据，按导入逻辑合并）。

## 三、主界面与对话

### 3.1 布局

- **左侧**：对话列表（可滚动）。底部有「**新建对话**」按钮。  
- **右侧**：当前对话的标签栏 + 聊天区域。  
- 左侧列表与右侧聊天区**独立滚动**，页面高度固定，不会整页滚动。

### 3.2 对话与标签

- **新建对话**：点击「新建对话」，在弹窗中填写「**话题**」（作为标题）并选择「**回复风格**」（仅对本对话生效）。创建后，该对话下的记忆会与其它对话隔离。  
- **切换对话**：在左侧列表点击某一对话即可切换；右侧会显示该对话的标签页。  
- **关闭标签**：右侧标签栏每个对话旁有「×」。点击后仅**关闭该标签页**，不会删除对话记录；再次从左侧点击该对话可重新打开。  
- **删除对话**：在左侧对某对话执行删除后，该对话及其**全部记忆**会被删除，且不可恢复。

### 3.3 聊天与记忆

- 在输入框输入内容后发送，助手会先根据当前问题**检索相关记忆**，再结合 LLM 生成回答。  
- 回复以**流式**方式输出，并支持 **Markdown** 实时渲染。  
- 对话结束后，插件会**自动从对话中提取值得长期记忆的内容**（事实、偏好、事件），并打上标签保存；若你设置了「自定义标签」，提取时会优先考虑这些标签。

## 四、设计理念：限制即保护

本插件的设计刻意做了一些「限制」，目的是**保护你的隐私与控制权**：

- **数据本地化**：对话内容、记忆、用户画像均保存在你的浏览器本地（IndexedDB），不上传到本插件或任何第三方服务器。  
- **按对话隔离**：每个对话的记忆独立存储；删除对话即删除该对话下的所有记忆，避免信息混杂与误用。  
- **可控的自动化**：虽然记忆是自动提取的，但你可以通过「自定义标签」「回复风格」「自定义说明」引导提取与回复方式。  
- **画像仅服务本人**：画像数据只用于优化你本人的对话与记忆体验，不用于训练或对外分享。

我们希望你**在知情、可控的前提下**使用本插件，并养成定期备份、合理分类的习惯。

## 五、持续学习：越用越贴合你

- **画像随使用积累**：你的提问会参与「近期常问」统计，提取出的记忆标签会参与「常用标签」与标签云，方便你了解自己的使用习惯。  
- **自定义标签与说明**：你填写的「自定义标签」和「自定义说明」会持续影响后续的**记忆提取**与**回答风格**，使助手越用越贴合你的分类习惯与表达偏好。  
- **建议用法**：  
  - 在设置中维护一批你常用的「自定义标签」（如工作、学习、生活、兴趣等）。  
  - 新建对话时尽量填写「话题」和「回复风格」。  
  - 定期在设置页查看「近期常问」「常用标签」与标签云，必要时调整自定义说明与标签，形成**学习与整理闭环**。

合理应用这些功能，可以让本插件成为你**长期、可持续**的学习与记忆外脑。

## 六、常见问题

- **扩展无法连接 LLM**：检查设置页的 API 地址、Key、模型名是否正确；若用 Ollama，确认本地已启动且地址可访问。  
- **记忆没有按对话分开**：请确认使用的是「新建对话」创建的对话；旧版本升级后，新对话会按对话隔离，旧数据可能需手动整理或导出后再导入。  
- **自定义标签不生效**：添加或修改标签后，需在设置页点击「**保存画像**」；提取记忆会在**下一次**对话自动提取时使用新标签。  
- **想清空或重置用户画像**：在设置页「用户画像」区域使用「清空自动统计」（只清常问与常用标签）、或「重置用户画像」（恢复全部为默认）。  
- **数据丢失**：重要记忆建议定期使用「导出记忆」备份；卸载或清除扩展数据会导致本地数据丢失。
