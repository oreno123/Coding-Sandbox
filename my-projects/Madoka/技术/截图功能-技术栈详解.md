# 区域截图功能 - 技术栈详解

## 功能实现概览

本功能允许用户在 Madoka 侧边栏中点击截图按钮后，**在目标网页上拖拽选择矩形区域**，将选区截图与文本一起发送给通义千问多模态模型（qwen-vl-plus），实现图文混合对话。

---

## 技术栈详解

### 1. 技术栈清单

| 技术 | 版本/说明 | 作用 |
|------|-----------|------|
| Chrome Extension Manifest V3 | MV3 | 浏览器扩展运行环境 |
| `chrome.tabs.captureVisibleTab` | Chrome API | 截取标签页可见区域（整页） |
| `chrome.runtime.sendMessage` / `chrome.tabs.sendMessage` | Chrome Messaging API | 侧边栏 ↔ Background ↔ Content Script 通信 |
| Canvas API `drawImage` | 原生 Web API | 按选区坐标裁剪截图 |
| `window.devicePixelRatio` | 原生 Web API | 高分屏下坐标与像素换算 |
| React 18 + TypeScript | 项目现有 | 侧边栏 UI 与状态管理 |
| 通义千问 Qwen-VL | qwen-vl-plus | 多模态视觉语言模型 |
| OpenAI 兼容 API 格式 | - | 多模态消息结构（text + image_url） |

---

### 2. 各技术作用与选型理由

#### 2.1 方案一：页面上选区 + 整页截图 + Canvas 裁剪

- **作用**：用户直接在网页上拖拽出矩形，所见即所得；Background 截整页，Content Script 用 Canvas 按选区裁剪。
- **选型理由**：Chrome 的 `captureVisibleTab` 不支持直接截取指定区域，只能先截整页再裁剪。在页面上选区更直观，坐标与视口一致，只需在裁剪时乘以 `devicePixelRatio`。

#### 2.2 `chrome.tabs.captureVisibleTab`

- **作用**：截取指定窗口中当前可见标签页的可见区域，返回 PNG 格式的 Data URL。
- **选型理由**：Chrome 官方截图 API，需 `host_permissions: ["<all_urls>"]`。在 Background 中调用可避免侧边栏场景下 `activeTab` 的「调用」校验失效问题。

#### 2.3 Content Script 选区覆盖层

- **作用**：在目标页面注入全屏 `position: fixed` 覆盖层，监听 `mousedown` / `mousemove` / `mouseup` 实现拖拽选区，左键松开后右键确认、Esc 取消。
- **选型理由**：选区必须在页面上下文中完成，以便坐标与视口一致；Content Script 可访问 `document` 和 `window.devicePixelRatio`。

#### 2.4 Canvas `drawImage` 裁剪

- **作用**：将整页截图按 `(left, top, width, height)` 裁剪，需乘以 `devicePixelRatio` 以匹配 `captureVisibleTab` 返回的高分屏像素。
- **选型理由**：`captureVisibleTab` 返回的图片尺寸 = 视口尺寸 × devicePixelRatio，选区坐标是 CSS 像素，裁剪时需换算。

#### 2.5 通义千问 Qwen-VL（qwen-vl-plus）

- **作用**：支持「文本 + 图片」输入的多模态大模型。
- **选型理由**：与现有 API 端点兼容，只需在请求中切换 `model` 字段。

---

### 3. 关键概念简明解释

- **Data URL**：将二进制数据（如图片）编码为 base64 字符串，格式 `data:image/png;base64,<base64>`，可直接作为 `img.src` 或 API 的 `image_url.url`。
- **devicePixelRatio**：设备像素比，如 2 表示 1 个 CSS 像素对应 2×2 物理像素；`captureVisibleTab` 返回的图片按物理像素，选区坐标按 CSS 像素，裁剪时需乘以该值。
- **host_permissions**：`<all_urls>` 授予对任意网页的访问权限，满足 `captureVisibleTab` 的权限要求，无需 `activeTab`。

---

### 4. 数据流简要说明

1. 用户点击截图按钮 → 侧边栏发送 `startRegionCapture` 给 Background
2. Background 获取 `lastFocusedWindow` 的 active tab，向 Content Script 发送 `showRegionSelector`
3. Content Script 显示选区覆盖层，用户左键拖拽选择 → 左键松开后右键确认，发送 `regionSelected`（含 rect）给 Background
4. Background 调用 `captureVisibleTab` 获取整页 Data URL，再发送 `cropScreenshot`（含 dataUrl、rect）给 Content Script
5. Content Script 用 Canvas 裁剪，发送 `croppedScreenshot`（含 dataUrl）给 Background
6. Background 通过 `pendingRegionCaptureResolve` 将结果回传给侧边栏
7. 侧边栏将 Data URL 存入 `attachedImages`，用户发送时走多模态 API

## 配置说明

| 配置项 | 位置 | 作用 |
|--------|------|------|
| `visionModel` | `src/shared/types.ts` → `DEFAULT_CONFIG` | 有截图时使用的视觉模型，默认 `qwen-vl-plus` |
| `host_permissions: ["<all_urls>"]` | `manifest.json` | 用于 `captureVisibleTab` |
| Content Script `matches: ["<all_urls>"]` | `manifest.json` | 确保目标页面加载选区逻辑 |
