# 在对话中显示被引用的页面 - 方案文档

## 问题
用户在使用 `@` 引用文件后，无法在对话中直观地看到哪些页面被引用了，缺乏引用确认的视觉反馈。

## 核心目标
让用户明确知道：**页面确实被引用了**，并且知道**引用了哪些页面**。

---

## 方案对比

### 方案 1: 在用户消息下方显示引用标签（推荐）

**设计思路**：
类似于 ChatGPT/Claude 的引用显示方式，在用户发送的消息下方，以标签形式展示被引用的页面。

**实现方式**：
- 修改 `Message` 组件，在用户消息（`role === 'user'`）中增加引用展示区域
- 从 `message` 对象中获取引用的页面信息（需要在发送时保存）
- 使用类似 `AttachedContextBar` 的 Chip 样式，但更小更简洁

**视觉效果**：
```
┌─────────────────────────────────────┐
│ 用户输入的消息内容                     │
├─────────────────────────────────────┤
│ 📎 GitHub | 📎 百度                  │  ← 引用标签行
└─────────────────────────────────────┘
```

**优点**：
- 直观，用户一眼就能看到引用了什么
- 与消息绑定，历史记录中也能看到
- 实现简单，复用现有组件

**缺点**：
- 需要修改 Message 类型，增加 contextRefs 字段
