# 截图点击放大功能实现详解

## 功能概述

Madoka 浏览器扩展中的截图点击放大功能允许用户在侧边栏中点击缩略图，在全屏查看器中查看大图。该功能采用了**内容脚本注入**的方式实现，而不是在侧边栏内直接放大。

---

## 核心架构

### 1. 数据流

```
用户点击缩略图
  ↓
Composer/Message 组件设置 viewingImage 状态
  ↓
ImageViewer 组件被渲染
  ↓
通过 chrome.tabs.sendMessage 发送消息到 Content Script
  ↓
Content Script 在网页上创建全屏覆盖层显示图片
  ↓
用户点击关闭（或按 Esc）
  ↓
Content Script 发送消息关闭查看器
  ↓
侧边栏 onClose 回调清除 viewingImage 状态
```

## 关键文件

### 1. `src/sidepanel/components/ImageViewer.tsx` - 侧边栏组件

**作用**：作为桥梁，将图片 URL 传递给 Content Script，在网页上显示查看器。

**核心代码**：
```typescript
export const ImageViewer = ({ imageUrl, onClose }: { 
  imageUrl: string | null, 
  onClose: () => void 
}) => {
  useEffect(() => {
    if (!imageUrl) return

    // 通过 content script 在页面中显示图片查看器
    chrome.tabs.query({ active: true, currentWindow: true }, (tabs) => {
      if (tabs[0]?.id) {
        chrome.tabs.sendMessage(
          tabs[0].id,
          { action: 'showImageViewer', imageUrl },
          () => {
            // 关闭侧边栏的查看器
            onClose()
          }
        )
      }
    })
  }, [imageUrl, onClose])

  // 不在侧边栏渲染任何内容
  return null
}
```

**设计理由**：
- 侧边栏空间有限，不适合全屏查看
- 在网页上显示可以充分利用屏幕空间
- 避免跨域 iframe 的安全限制

---

### 2. `src/content/image-viewer.ts` - Content Script 图片查看器

**作用**：在网页上创建全屏覆盖层，显示大图。

**核心函数**：

#### `createImageViewer(imageUrl: string): HTMLDivElement`

创建全屏覆盖层，包括：
- **覆盖层**：`position: fixed; inset: 0;` 全屏半透明黑色背景
- **图片容器**：居中显示，最大宽高 95vw/95vh
- **图片元素**：`object-fit: contain` 保持比例
- **关闭按钮**：右上角 SVG 图标

**关键样式**：
```css
overlay.style.cssText = `
  position: fixed;
  inset: 0;
  background: rgba(0, 0, 0, 0.5);
  z-index: 2147483647;  // 最高层级
  display: flex;
  align-items: center;
  justify-content: center;
  cursor: pointer;
  backdrop-filter: blur(4px);
`

img.style.cssText = `
  max-width: 100%;
  max-height: 95vh;
  border-radius: 8px;
  box-shadow: 0 25px 50px -12px rgba(0, 0, 0, 0.5);
  border: 2px solid rgba(255, 255, 255, 0.1);
  object-fit: contain;
`
```

**交互逻辑**：
```typescript
// 点击图片本身时阻止关闭
img.addEventListener('click', (e) => {
  e.stopPropagation()
})

// 点击覆盖层（图片外部区域）关闭
overlay.addEventListener('click', closeViewer)

// 点击关闭按钮关闭
closeBtn.addEventListener('click', (e) => {
  e.stopPropagation()
  closeViewer()
})

// 淡出动画
const closeViewer = () => {
  overlay.style.opacity = '0'
  overlay.style.transition = 'opacity 0.2s'
  setTimeout(() => {
    overlay.remove()
    state.viewerElement = null
    state.imageUrl = null
  }, 200)
}
```

#### `showImageViewer(imageUrl: string)`

显示图片查看器，如果已有查看器则先销毁。

#### `closeImageViewer()`

关闭图片查看器，清理 DOM 元素和状态。

### 3. `src/content/index.ts` - Content Script 消息处理

**作用**：监听来自侧边栏的消息，调用图片查看器。

**核心代码**：
```typescript
// 显示图片查看器
if (request.action === 'showImageViewer') {
  ;(async () => {
    try {
      const imageUrl = request.imageUrl as string
      showImageViewer(imageUrl)
      sendResponse({ success: true })
    } catch (e) {
      sendResponse({ success: false, error: (e as Error).message })
    }
  })()
  return true
}

// 关闭图片查看器
if (request.action === 'closeImageViewer') {
  closeImageViewer()
  sendResponse({ success: true })
  return true
}
```

## 技术亮点

### 1. **为什么使用 Content Script 而不是侧边栏内放大？**

**优点**：
- ✅ **全屏显示**：充分利用屏幕空间，不受侧边栏宽度限制
- ✅ **避免跨域问题**：侧边栏是 `chrome-extension://` 协议，图片可能是 `data:` 或其他协议
- ✅ **更好的用户体验**：类似桌面应用的全屏查看体验
- ✅ **性能优化**：大图片不会在侧边栏中频繁渲染

**缺点**：
- ❌ 需要注入 Content Script，增加复杂性
- ❌ 需要处理消息通信

---

### 2. **为什么使用最高 z-index？**

```typescript
overlay.style.zIndex = '2147483647'  // 2^31 - 1
```

**理由**：
- 确保覆盖层在所有网页元素之上
- 避免被网页的固定定位元素（如导航栏）遮挡
- Chrome 扩展的 Content Script 与网页共享 z-index 上下文

---

### 3. **图片适配策略**

```typescript
img.style.objectFit = 'contain'
img.style.maxWidth = '100%'
img.style.maxHeight = '95vh'
```

**效果**：
- 保持图片原始比例
- 适应不同屏幕尺寸
- 留出关闭按钮的空间（95vh 而非 100vh）

---

### 4. **交互细节**

#### 阻止图片点击冒泡
```typescript
img.addEventListener('click', (e) => {
  e.stopPropagation()
})
```
**理由**：点击图片时不关闭查看器，只有点击背景才关闭

#### 淡出动画
```typescript
overlay.style.opacity = '0'
overlay.style.transition = 'opacity 0.2s'
setTimeout(() => {
  overlay.remove()
  state.viewerElement = null
  state.imageUrl = null
}, 200)
```
**理由**：平滑过渡，提升用户体验

####  backdrop-filter 毛玻璃
```css
backdrop-filter: blur(4px);
```
**理由**：现代 UI 设计，模糊背景突出图片

## 总结

截图点击放大功能采用了**Content Script 注入**的方式，在网页上创建全屏覆盖层显示大图。这种设计避免了侧边栏空间限制，提供了更好的全屏查看体验。核心实现包括：

1. **侧边栏桥接**：`ImageViewer` 组件通过消息通信通知 Content Script
2. **全屏覆盖层**：使用最高 z-index 确保在所有元素之上
3. **图片适配**：`object-fit: contain` 保持比例，适应不同屏幕
4. **交互优化**：点击背景关闭、淡出动画、阻止图片点击冒泡
5. **状态管理**：React 状态驱动，组件化设计

整个实现简洁高效，用户体验流畅。
