# Writing Style Guide (写作风格指南)

本指南定义了生成软件工程作业报告的写作规范。请严格遵守以下规则。

## 核心原则

1.  **语言风格**
    *   **平实描述**：使用平实、具有描述性的语言。
    *   **避免修辞**：不要使用过多的修辞手法或复杂的转折句。
    *   **少用格式**：尽量减少Markdown文本格式（如粗体、斜体）的使用，保持纯文本的可读性。

2.  **绝对禁止**
    *   **禁止括号说明**：绝对不要使用括号来进行补充说明或解释（如 `(例如...)` 或 `(i.e. ...)`）。如果需要解释，请将其融入句子结构中。
    *   **禁止表格**：不要使用Markdown表格来展示数据或对比。请使用文本段落或列表进行描述。

3.  **句子结构**
    *   **允许长句**：可以使用较长的句子来完整描述一件事物，不必为了追求简洁而把句子切分得过于琐碎。
    *   **逻辑宽容度**：不必追求极致的结构化或严密的逻辑性。允许出现少量逻辑不够清晰但仍清晰可读的内容，还原真实的学生作业感。

4.  **内容详略**
    *   **简要介绍为主**：对于每个知识点或回答，进行简要明确的介绍。
    *   **适度展开**：可以包含少量的详细介绍，但不要对所有内容都进行长篇大论的详细描述。
    *   **平衡**：注意平衡文本量和内容覆盖面，既不要过于简略，也不要冗余。

5.  **语言**
    *   **中文为主**：主要使用中文撰写。
    *   **英文辅助**：在提及专业术语或必要时，可以使用英文进行解释（注意不要用括号，可以用“即...”或直接使用英文术语）。

6.  **输出格式规范**
    *   **Markdown (默认)**
        *   使用标准Markdown语法。
        *   层级标题清晰（#）。
    *   **LaTeX**
        *   使用标准 `article` 文档类。
        *   引入 `ctex` 宏包支持中文。
        *   仅引入必要的宏包（如 `geometry` 调整页边距），避免过度复杂的模板。
        *   **禁止使用 tabular 环境**（严格遵守不使用表格的规定）。
        *   **禁止在正文中使用括号**（代码或数学公式除外）。
