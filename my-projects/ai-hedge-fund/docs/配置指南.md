# AI Hedge Fund 应用配置指南

本文档记录 AI Hedge Fund 应用的完整配置过程，供后续部署或问题排查参考。

---

## 一、环境要求

| 依赖 | 版本要求 | 说明 |
|------|----------|------|
| Node.js | 18+ | 含 npm，用于前端构建 |
| Python | 3.11+ | 建议 3.11，3.13 需额外处理（见下文） |
| Poetry | 最新版 | Python 包管理工具 |

---

## 二、配置步骤

### 1. 添加 app 包初始化文件

**问题**：`app` 目录缺少 `__init__.py`，导致 `app.backend.main` 导入失败。

**处理**：在 `app/` 目录下新建 `__init__.py` 文件（可为空或仅含注释）。

```
app/
├── __init__.py      ← 新增
├── backend/
├── frontend/
└── ...
```

---

### 2. 环境变量配置

**步骤**：

1. 在项目根目录复制 `.env.example` 为 `.env`：
   ```bash
   cp .env.example .env
   ```

2. 编辑 `.env`，至少配置一个 LLM API Key：
   ```
   # 必填其一（用于运行 AI 分析）
   OPENAI_API_KEY=sk-xxx
   # 或 GROQ_API_KEY、ANTHROPIC_API_KEY、DEEPSEEK_API_KEY 等

   # 可选（分析 AAPL/GOOGL/MSFT/NVDA/TSLA 以外的股票时需配置）
   FINANCIAL_DATASETS_API_KEY=your-key
   ```

---

### 3. Python 依赖安装

**步骤**：

```bash
cd D:\desktop\ai-hedge-fund
poetry install
```

**Python 3.13 兼容性**：若使用 Python 3.13，需修改 `pyproject.toml` 中 numpy 版本：

```toml
# 原配置（Python 3.13 下可能编译失败）
numpy = "^1.24.0"

# 修改为（使用预编译 wheel，避免源码编译）
numpy = "^2.0.0"
```

修改后执行：

```bash
poetry lock
poetry install
```

**ormsgpack 导入错误**（Windows 下可能出现）：

```bash
poetry run pip install --force-reinstall ormsgpack
```

---

### 4. 前端依赖安装

```bash
cd app/frontend
npm install
```

---

### 5. 前端 TypeScript 修复（可选）

若 `npm run build` 报错，可按以下方式处理：

| 问题 | 处理 |
|------|------|
| Layout 导入大小写 | `./components/layout` → `./components/Layout` |
| Layout 的 children 必填 | 将 `LayoutProps` 中 `children` 改为可选 `children?: ReactNode` |
| 未使用变量/参数 | 加 `_` 前缀或删除，如 `provider` → `_provider` |
| sidebar.tsx ref 类型 | 对 Slot 的 ref 使用 `ref={ref as any}` 绕过类型检查 |

## 三、启动应用

### 方式 A：一键启动（推荐）

**Windows**：
```cmd
cd D:\desktop\ai-hedge-fund\app
run.bat
```

**Mac/Linux**：
```bash
cd ai-hedge-fund/app
./run.sh
# 或
bash run.sh
```

脚本会检查依赖、安装缺失包、启动前后端并打开浏览器。

### 方式 B：手动启动

**终端 1 - 后端**：
```bash
cd D:\desktop\ai-hedge-fund
poetry run uvicorn app.backend.main:app --reload --host 127.0.0.1 --port 8000
```

**终端 2 - 前端**：
```bash
cd D:\desktop\ai-hedge-fund\app\frontend
npm run dev
```

## 六、常见问题
image.png
### 1. Poetry 未安装

```bash
# Windows (PowerShell)
(Invoke-WebRequest -Uri https://install.python-poetry.org -UseBasicParsing).Content | python -

# Mac/Linux
curl -sSL https://install.python-poetry.org | python3 -
```

### 2. uvicorn 找不到

在项目根目录执行：

```bash
poetry install
poetry run python -c "import uvicorn; print('OK')"
```

### 3. 端口被占用

- 8000：后端
- 5173：前端

可结束占用进程，或修改 `run.bat` / `run.sh` 中的端口配置。

### 4. 数据库位置

SQLite 数据库默认路径：`app/backend/hedge_fund.db`，首次 API 调用时自动创建。
