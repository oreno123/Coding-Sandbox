# 图标更新后未生效问题修复计划

## 问题分析

### 当前状态
- **源图标位置**：`public/icons/` 目录（icon16.png, icon48.png, icon128.png）
- **构建后位置**：`dist/public/icons/` 目录
- **manifest.json 路径**：`public/icons/icon16.png` 等

### 可能原因

1. **浏览器扩展缓存**：Chrome 会缓存已加载的扩展资源
2. **构建缓存**：Vite 构建时可能使用了缓存的旧文件
3. **扩展未重新加载**：构建后没有在扩展管理页面刷新扩展
4. **dist 目录残留**：旧的 dist 文件没有被清理

## 解决步骤

### 步骤 1：清理构建缓存
删除 `dist` 目录和 Vite 缓存，确保完全重新构建。

### 步骤 2：重新构建
运行 `npm run build` 重新构建项目。

### 步骤 3：重新加载扩展
在 Chrome 扩展管理页面（`chrome://extensions/`）：
1. 点击扩展卡片上的刷新按钮
2. 或者移除扩展后重新加载

### 步骤 4：清除浏览器缓存（可选）
如果仍然不生效，清除浏览器缓存或使用隐身模式测试。

## 验证方法
1. 检查 `dist/public/icons/` 中的图标文件是否已更新（查看文件修改时间）
2. 在浏览器中查看扩展图标是否变化
