# Madoka UI 优化计划

## 当前UI分析

### 技术栈
- React 18 + TypeScript
- TailwindCSS
- Framer Motion (动画)
- 主题系统: 支持 Light/Dark 模式

### 现有设计特点
- Cursor-style 极简设计
- 纯黑白灰配色
- 流体动画效果
- 沉浸式体验

---

## 优化建议与实施计划

### 1. 视觉层次优化

#### 1.1 消息气泡设计升级
**当前问题**: 消息气泡样式较为基础，缺乏视觉层次

**优化方案**:
- 添加渐变背景效果
- 优化圆角和阴影
- 区分用户/AI消息视觉权重
- 添加消息状态指示器(发送中、已送达)

**实施步骤**:
1. 更新 `Message.tsx` 组件样式
2. 在 `index.css` 中添加新的消息气泡样式变量
3. 添加消息状态动画

#### 1.2 输入区域(Composer)视觉提升
**当前问题**: 输入框与工具栏视觉融合度不够

**优化方案**:
- 添加玻璃拟态(Glassmorphism)效果
- 优化工具栏图标视觉统一性
- 添加输入框聚焦时的光晕效果
- 优化 BlockSelector 的视觉呈现

**实施步骤**:
1. 重构 `Composer.tsx` 样式
2. 更新 `index.css` 中 composer 相关样式
3. 优化工具按钮的 hover/active 状态

## 技术实现注意事项

1. **保持现有架构**: 继续使用 TailwindCSS + CSS 变量的方案
2. **动画性能**: 使用 Framer Motion 的 `layout` 属性优化布局动画
3. **主题兼容性**: 所有样式变更需同时适配 Light/Dark 模式
4. **可访问性**: 确保颜色对比度符合 WCAG 标准
5. **渐进增强**: 新功能应优雅降级
