# 板块设计的用途说明

## 什么是板块（Block）

板块是记忆的分类标签，用于将不同类型的记忆分组管理。例如：
- **学习板块** - 学习相关的记忆（正在学的技能、知识点等）
- **工作板块** - 工作相关的记忆（项目、职业规划等）
- **生活板块** - 日常生活（饮食、运动、习惯等）
- **感情板块** - 人际关系、情感相关
- **职业发展板块** - 长期职业规划、目标等

## 板块设计的核心用途

### 1. 精准携带记忆上下文

**场景**：用户想基于某类记忆提问

**例子**：
```
用户：帮我复习一下之前学的 TypeScript 内容

操作：
1. 用户在板块选择器中勾选「学习板块」
2. 发送消息
3. 系统只携带「学习板块」的记忆
4. LLM 基于这些记忆回答

效果：
- 不携带无关的生活、工作记忆
- 上下文更精准，回答更相关
- 减少 token 消耗
```

### 2. 避免记忆过载

**问题**：
- 如果每次都携带所有记忆，token 消耗巨大
- 无关记忆会干扰 LLM 的回答

**解决**：
- 通过板块筛选，只携带相关记忆
- 默认只携带最近3条 + 用户画像
- 选择板块后携带该板块最多10条记忆

### 3. 结构化记忆管理

**对比**：

| 无板块 | 有板块 |
|--------|--------|
| 记忆混乱堆积 | 分类清晰 |
| 难以查找 | 按板块浏览 |
| 全部携带或全部丢弃 | 按需携带 |

**Obsidian 同步效果**：
```
MadokaMemory/
├── 学习板块/
│   ├── TypeScript基础.md
│   └── React进阶.md
├── 工作板块/
│   └── 项目经验.md
└── 生活板块/
    └── 健康习惯.md
```

### 4. 支持多维度对话

**场景1：学习咨询**
```
选择：学习板块
提问：我之前学到哪了？
→ 携带学习记忆，回答学习进度
```

**场景2：职业规划**
```
选择：职业发展板块
提问：根据我的目标，下一步该做什么？
→ 携带职业记忆，给出职业建议
```

**场景3：日常闲聊**
```
不选择板块
提问：今天天气怎么样？
→ 只携带用户画像 + 最近3条记忆
→ 知道用户是谁，但不会携带过多无关信息
```

## 实际使用流程

### 首次使用
1. 正常对话，系统自动保存记忆（带板块标签）
2. 记忆管理页面可以看到不同板块的记忆

### 后续使用
1. 想基于某类记忆提问时
2. 在输入框上方点击选择对应板块
3. 输入问题发送
4. Madoka 会基于选中板块的记忆回答

### 不选择板块
- 默认携带用户画像（姓名、身份等）
- 默认携带最近3条记忆
- 适合日常闲聊或不需要特定背景的问题

## 板块 vs 标签的区别

| 特性 | 板块（Block） | 标签（Tag） |
|------|--------------|------------|
| 层级 | 一级分类 | 细粒度标记 |
| 用途 | 记忆分组管理 | 关键词检索 |
| 数量限制 | 最多10个 | 无限制 |
| 用户操作 | 主动选择携带 | 系统自动提取 |

## 总结

板块设计的核心价值：
1. **精准** - 只携带相关记忆，提高回答质量
2. **高效** - 减少 token 消耗，降低成本
3. **有序** - 结构化记忆，便于管理
4. **灵活** - 按需选择，适应不同场景

简单说：板块让用户能够"按需取用"记忆，而不是每次都把所有记忆都塞给 LLM。
