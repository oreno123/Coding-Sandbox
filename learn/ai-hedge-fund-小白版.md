## 零、这个项目到底在干嘛？

想象你有一个投资顾问团队：

- 有一个专门看K线图的技术分析师
- 有一个专门看财报的基本面分析师
- 有一个专门算公司值多少钱的估值师
- 有一个专门看新闻和舆情的情绪分析师
- 还有12个"角色扮演"投资大师的AI（假装自己是巴菲特、林奇等等）
- 最后有一个风控经理和一个拍板做决定的组合经理

你告诉他们："帮我分析苹果（AAPL）和微软（MSFT）这两只股票该买还是该卖。"

所有分析师同时开始干活（并行），每个人给出自己的结论：
- 技术师说："苹果趋势向上，我 70% 确信该买"
- 基本面师说："苹果利润很好，我 60% 确信该买"
- 估值师说："苹果现在有点贵，我 55% 确信该卖"

所有人分析完后，风控经理说："根据风险计算，苹果最多只能买 5000 美元的。"

最后组合经理综合所有人的意见，做出决定："买 30 股苹果，微软持有不动。"

**这就是整个系统干的事。注意：它不会真的下单，只是给出建议。**

## 一、先搞懂几个基础概念

### 1.1 股票市场的基本概念

**股票代码（Ticker）**：每只股票有一个简称。比如 AAPL = 苹果公司，MSFT = 微软，NVDA = 英伟达。

**做多（买/买涨）**：你觉得股票会涨，先买入，等涨了再卖掉赚差价。

**做空（卖空）**：你觉得股票会跌，先借股票卖掉，等跌了再低价买回来还掉，赚差价。

**持仓**：你现在手里持有的股票。

**组合（Portfolio）**：你所有持仓 + 现金的总和。

### 1.2 三个信号

每个分析师最后的结论只有三种：

| 信号 | 英文 | 含义 |
|------|------|------|
| 看涨 | bullish | 觉得会涨，建议买 |
| 看跌 | bearish | 觉得会跌，建议卖 |
| 中立 | neutral | 不确定，建议观望 |

每个信号还带一个**置信度（confidence）**，0-100分，表示分析师有多确信自己的判断。90分 = 非常确定，30分 = 不太确定。

### 1.3 什么是 LLM

LLM = 大语言模型，就是 ChatGPT、DeepSeek 这类 AI。在这个项目里，有些分析师是用 AI 来做判断的，有些是用纯数学公式算的。

### 2.2 怎么让 18 个分析师同时干活？

这个项目用一个叫 **LangGraph** 的 Python 库来编排。你可以把它想象成一个流水线管理员：

1. 管理员喊"开始"（start_node）
2. 18 个分析师同时开始干活（fan-out，像扇子一样展开）
3. 管理员等所有人干完（fan-in，像扇子收回来）
4. 把结果传给风控经理
5. 风控经理把结果传给组合经理

你不需要写任何多线程代码，LangGraph 自动帮你处理。

### 2.3 所有人的工作成果放在哪？

放在一个共享的"公告板"上（代码里叫 `state`）：

```
公告板（state）
├── data
│   ├── tickers: ["AAPL", "MSFT"]           ← 要分析哪些股票
│   ├── start_date: "2024-01-01"            ← 从什么时候开始看数据
│   ├── end_date: "2024-03-01"              ← 到什么时候
│   ├── portfolio: {cash: 100000, ...}      ← 你有多少钱和持仓
│   └── analyst_signals:                     ← 每个分析师的结论
│       ├── "技术分析师": {AAPL: {信号, 置信度}, MSFT: {...}}
│       ├── "基本面分析师": {AAPL: {信号, 置信度}, MSFT: {...}}
│       ├── "估值师": {AAPL: {信号, 置信度}, MSFT: {...}}
│       ├── "风控经理": {AAPL: {最大仓位, 当前价格}, MSFT: {...}}
│       └── ...
├── messages: [各分析师的详细分析报告]
└── metadata: {用什么AI模型, 是否显示推理过程}
```

每个分析师干完活就把自己的结论写到 `analyst_signals` 里，后面的人就能看到。

## 三、技术分析师 — 看K线图的人

**这个模块完全用数学公式算，不用 AI，是最靠谱的部分。**

技术分析师看的是历史价格走势，试图从"图"中找到规律。

### 3.1 它用了 5 种分析方法

就像看病要拍 5 种片子一样，技术分析师用 5 种不同的"镜头"看同一只股票，然后综合判断：

| 方法 | 一句话解释 | 权重（重要性） |
|------|-----------|-------------|
| 趋势跟踪 | 顺着大方向走，"涨就买，跌就卖" | 25% |
| 动量 | 看最近跑得快不快，"强者恒强" | 25% |
| 均值回归 | 涨太多了会回调，跌太多了会反弹 | 20% |
| 波动率 | 看市场是太安静了（要变天）还是太疯狂了（要冷静） | 15% |
| 统计套利 | 用高深统计判断股价是"有规律"还是"瞎走的" | 15% |

### 3.2 方法一：趋势跟踪（权重 25%）

**生活类比**：你站在河边看水流。如果水面所有漂浮物都在往同一个方向漂，你就知道水流方向了。

**具体做法**：

画 3 条不同周期的"平均线"（技术上叫 EMA，指数移动平均线）：

- **短期线**（8天）：最近 8 天的平均价格，反应快
- **中期线**（21天）：最近 21 天的平均价格
- **长期线**（55天）：最近 55 天的平均价格，反应慢

**为什么用平均线？** 因为每天的股价上蹿下跳，平均线能帮你"抹平"这些噪音，看到真正的大方向。

**判断规则**：

```
如果 短期线 > 中期线 > 长期线 → 涨势确立 → 看涨
如果 短期线 < 中期线 < 长期线 → 跌势确立 → 看跌
如果 乱七八糟的               → 没有明确趋势 → 中立
```

用画图理解（假设股价是 100，105，103，108，110...）：

```
价格
110 ────────────── 长期线（慢，反应迟钝）
105 ──────── 中期线（中等速度）
103 ──── 短期线（快，紧跟价格）
100
```

当短期线在中期线上方，中期线在长期线上方，说明"短、中、长期都在涨"。

**置信度怎么算？** 用一个叫 ADX 的指标。ADX 衡量趋势有多"强"：

- ADX > 25：趋势很强，我很有信心
- ADX < 20：基本没有趋势，没什么信心
- 置信度 = ADX / 100

### 3.3 方法二：均值回归（权重 20%）

**生活类比**：弹簧原理。你把弹簧拉太长，松手后它会弹回来。股价也一样——涨太多了会跌回来，跌太多了会涨回来。

**具体指标**：

#### Z-score（标准化偏离程度）

```
Z = (当前价格 - 过去50天平均价) / 过去50天价格的标准差
```

- Z = 0：当前价格刚好在平均水平
- Z = +2：比平均水平高了 2 个标准差，可能涨过头了
- Z = -2：比平均水平低了 2 个标准差，可能跌过头了

**什么是标准差？** 可以理解为"正常波动范围"。如果某只股票每天正常波动 ±2%，那 2 个标准差大约就是 ±4%。现在跌了 5%，就是"不正常"了。

#### 布林带

布林带就是在平均线的上方和下方各画一条线：

```
     ═══════════ 上轨 = 均线 + 2×标准差
     ─────────── 均线 = 20天平均价
     ═══════════ 下轨 = 均线 - 2×标准差
```

正常情况下，股价 95% 的时间都在这两条轨道之间运行。

如果股价跌到下轨附近甚至穿出去了 → 超卖，可能要反弹
如果股价涨到上轨附近甚至穿出去了 → 超买，可能要回调

**判断规则**：

```
Z < -2 且 股价在布林带最底部 → 超卖了，该涨了 → 看涨
Z > +2 且 股价在布林带最顶部 → 超买了，该跌了 → 看跌
其他情况                     → 看不出什么 → 中立
```

### 3.4 方法三：动量（权重 25%）

**生活类比**：一辆车如果一直在加速，短时间内不太可能突然倒退。动量就是看这辆车最近跑得快不快。

**具体做法**：

看过去 1 个月、3 个月、6 个月的收益率（涨了还是跌了），然后加权平均：

```
动量分数 = 1个月涨跌 × 40% + 3个月涨跌 × 30% + 6个月涨跌 × 30%
```

1 个月权重最大（40%），因为最近的走势最有参考价值。

**还要看成交量**：如果价格上涨的同时交易量也放大了（比 21 天平均交易量高），说明很多人在买，信号更可靠。

```
动量分数 > 5% 且 放量 → 上涨动力很足 → 看涨
动量分数 < -5% 且 放量 → 下跌动力很足 → 看跌
其他                  → 中立
```

### 3.5 方法四：波动率分析（权重 15%）

**生活类比**：暴风雨前常常异常平静。股市也一样——如果一段时间波动特别小，可能是在蓄力准备大变；如果波动特别大，可能即将回归平静。

**波动率怎么算？**

```
日收益率 = (今天价格 - 昨天价格) / 昨天价格
波动率 = 最近21天日收益率的标准差
年化波动率 = 日波动率 × √252（一年大约252个交易日）
```

**波动率体制** = 当前波动率 / 过去 63 天平均波动率

- 体制 < 0.8：当前波动率比平时低很多 → 异常平静，可能要变天
- 体制 > 1.2：当前波动率比平时高很多 → 太疯狂了，可能要回归正常

**判断规则**：

```
波动率异常低 且 统计上显著低于正常 → 看涨（预期波动要放大，通常伴随上涨）
波动率异常高 且 统计上显著高于正常 → 看跌（预期波动要缩小，通常伴随回调）
其他                              → 中立
```

### 3.6 方法五：Hurst 指数（权重 15%）

**这是最高深的一个方法。**

**生活类比**：你在看一只蚂蚁走路。如果它完全随机乱走（醉汉走路），你无法预测它下一步去哪。但如果它有目的地在走（比如朝食物方向），你可以预测它会继续朝那个方向走。

Hurst 指数就是判断股价是"随机乱走的"还是有"记忆"的：

```
H < 0.5 → 有"回归"记忆：偏离后会回来（像弹簧）
H = 0.5 → 完全随机：没有规律（像醉汉走路）
H > 0.5 → 有"趋势"记忆：涨了还继续涨，跌了还继续跌
```

**怎么算的？** 核心思想是看价格变化在不同时间尺度上的表现：

1. 看价格间隔 2 天的差异
2. 看价格间隔 3 天的差异
3. ... 一直看到间隔 20 天的差异
4. 对这些差异取对数，画一条线拟合
5. 这条线的斜率就是 Hurst 指数

**判断规则**：

```
H < 0.4 且 分布偏右（正偏度） → 价格有回归倾向且偏向正 → 看涨
H < 0.4 且 分布偏左（负偏度） → 价格有回归倾向且偏向负 → 看跌
其他                         → 中立
```

### 3.7 五个方法怎么合成一个结论？

**加权投票**：

把每个方法的结论转成数字：
- 看涨 = +1
- 中立 = 0
- 看跌 = -1

然后加权求和：

```
最终分数 = (
    趋势的分数 × 25% × 趋势的置信度
  + 均值回归的分数 × 20% × 均值回归的置信度
  + 动量的分数 × 25% × 动量的置信度
  + 波动率的分数 × 15% × 波动率的置信度
  + 统计套利的分数 × 15% × 统计套利的置信度
) / 总权重

如果 最终分数 > +0.2  → 看涨
如果 最终分数 < -0.2  → 看跌
如果 在 -0.2 到 +0.2 之间 → 中立
```

**为什么阈值是 0.2？** 如果设成 0，那任何微小的倾向都会触发信号，太容易误判。设成 0.2，意味着要"比较确定"才会发出信号，减少误判。

**举个例子**：

```
趋势跟踪：看涨(+1), 置信度 0.7, 权重 25%
均值回归：看跌(-1), 置信度 0.6, 权重 20%
动量：    看涨(+1), 置信度 0.8, 权重 25%
波动率：  中立(0),   置信度 0.5, 权重 15%
统计套利：看涨(+1), 置信度 0.5, 权重 15%

分子 = (+1×0.25×0.7) + (-1×0.20×0.6) + (+1×0.25×0.8) + (0×0.15×0.5) + (+1×0.15×0.5)
     = 0.175 - 0.12 + 0.20 + 0 + 0.075
     = 0.33

分母 = 0.25×0.7 + 0.20×0.6 + 0.25×0.8 + 0.15×0.5 + 0.15×0.5
     = 0.175 + 0.12 + 0.20 + 0.075 + 0.075
     = 0.645

最终分数 = 0.33 / 0.645 = 0.51

0.51 > 0.2 → 看涨，置信度 = 51%
```

## 四、基本面分析师 — 看财报的人

**纯数学打分，不用 AI。**

基本面分析师看的是公司的"体检报告"——赚不赚钱、负不负债、长得快不快。

### 4.1 四个检查项目

#### 检查一：盈利能力（公司赚不赚钱？）

| 指标 | 含义 | 达标线 | 达标+1分 |
|------|------|--------|---------|
| ROE（净资产收益率） | 股东投 100 块，公司一年能赚多少 | > 15% | ✓ |
| 净利率 | 每 100 块收入，能剩多少利润 | > 20% | ✓ |
| 营业利润率 | 扣除日常运营成本后还剩多少 | > 15% | ✓ |

- 3 项中 ≥ 2 项达标 → 看涨
- 0 项达标 → 看跌
- 1 项达标 → 中立

**大白话**：如果一家公司 ROE 是 25%（股东投 100 块赚 25 块），净利率 22%（每卖 100 块东西净赚 22 块），营业利润率 18%——三项都达标，盈利能力很强。

#### 检查二：成长性（公司长得快不快？）

| 指标 | 含义 | 达标线 |
|------|------|--------|
| 营收增长 | 今年比去年多赚了多少 | > 10% |
| 盈利增长 | 利润增长了多少 | > 10% |
| 账面价值增长 | 公司的净资产增长了多少 | > 10% |

同样的打分规则：≥ 2 项达标看涨，0 项看跌。

#### 检查三：财务健康（公司会不会倒闭？）

| 指标 | 含义 | 达标线 | 大白话 |
|------|------|--------|--------|
| 流动比率 | 流动资产 / 流动负债 | > 1.5 | 手里的现金够不够还短期债务 |
| 负债权益比 | 总负债 / 股东权益 | < 0.5 | 借的钱比自己的钱少不多 |
| 自由现金流质量 | 自由现金流/股 > 盈利/股 × 0.8 | > 0.8 | 赚的钱是不是真金白银（不是纸上富贵） |

#### 检查四：估值（现在买贵不贵？）

这个是反着来的——指标越高越不好：

| 指标 | 含义 | 太贵的线 |
|------|------|---------|
| P/E（市盈率） | 股价 / 每股利润 | > 25 |
| P/B（市净率） | 股价 / 每股净资产 | > 3 |
| P/S（市销率） | 股价 / 每股营收 | > 5 |

**大白话**：P/E = 25 意味着你花 25 块买 1 块的年利润，要 25 年才回本。P/E > 25 就有点贵了。

- ≥ 2 项偏贵 → 看跌（太贵了，别买）
- 0 项偏贵 → 看涨（还便宜）
- 1 项 → 中立

### 4.2 四个检查怎么合成？

**少数服从多数**：4 个检查中，看涨最多的就听谁的。

```
比如：
  盈利能力 → 看涨
  成长性   → 看涨
  财务健康 → 中立
  估值     → 看跌

看涨有 2 票，看跌 1 票，中立 1 票
看涨最多 → 最终结论：看涨

置信度 = 看涨票数 / 总票数 × 100 = 2/4 × 100 = 50%
```

## 五、估值师 — 算公司值多少钱的人

**纯数学，不用 AI。用 4 种方法算公司"应该"值多少钱，然后和"现在"的价格比。**

**核心思路**：如果算出来公司应该值 1000 亿，但现在市值只有 800 亿 → 被低估了，该买。反过来就是被高估了，该卖。

### 5.1 方法一：DCF 现金流折现（权重 35%）

**最经典的估值方法。**

**生活类比**：你开了一家奶茶店，每年净赚 10 万。你想知道这家店"值多少"。

思路：把未来每年能赚的钱都算出来，然后打个折（因为明年的 10 万不如今天的 10 万值钱——通胀+机会成本），加在一起就是店铺的价值。

**具体做法 — 三阶段模型**：

```
阶段 1（第1-3年）：高速增长期
  假设奶茶店每年多赚 10%（增长来自扩张新店）

阶段 2（第4-7年）：过渡期
  增长放缓到 5%（市场开始饱和）

阶段 3（永续）：稳定期
  增长率降到 3%（差不多跟 GDP 一样）
```

**折现率**（打多少折）是用 WACC 算的，详见下文。

**三情景分析**：因为未来不确定，所以算三次：

| 情景 | 假设 | 概率 |
|------|------|------|
| 熊市（悲观） | 增长减半，折现率提高 20% | 20% |
| 基准（正常） | 按当前数据算 | 60% |
| 牛市（乐观） | 增长 × 1.5，折现率降低 10% | 20% |

```
最终估值 = 熊市估值 × 20% + 基准估值 × 60% + 牛市估值 × 20%
```

概率最高的是基准（60%），所以基准情景对最终估值影响最大。

#### WACC（加权平均资本成本）是什么？

就是"折现率"——你要求的最少年化回报率。

**生活类比**：你有两个选择，把钱存银行拿 4.5% 利息，或者投资奶茶店。奶茶店有风险，你至少要赚 10% 才觉得值得。这个 10% 就是你的折现率。

**公司怎么算 WACC？**

```
WACC = 股东要求回报率 × 股东占比 + 债权人要求回报率 × 债权人占比 × (1 - 税率)

其中：
股东要求回报率 = 4.5%（国债利率） + 1.0（市场风险系数）× 6%（市场风险溢价）= 10.5%
债权人要求回报率 = 国债利率 + 信用利差（利息覆盖率越高，信用越好，利率越低）

WACC 范围：6% - 20%
```

### 5.2 方法二：Owner Earnings 所有者盈余（权重 35%）

**巴菲特最喜欢的估值方法。**

**问题**：财报上的"净利润"不等于你真正能拿走的钱。因为有些钱你必须花出去（比如修设备、补充库存）。

**巴菲特的公式**：

```
所有者盈余 = 净利润 + 折旧摊销 - 维持性资本支出 - 营运资本变化
```

逐项解释：

| 项目 | 含义 | 为什么要加/减 |
|------|------|-------------|
| 净利润 | 财报上的利润 | 起点 |
| +折旧摊销 | 设备老化扣的钱 | 这不是真的花出去的现金，只是会计记账 |
| -维持性CapEx | 为了维持现有业务必须花的钱 | 这是真的要花的钱 |
| -营运资本变化 | 库存、应收账款的变化 | 如果库存增加了，钱被压住了 |

**维持性CapEx怎么估算？**

用三种方法算，取中位数：

| 方法 | 思路 |
|------|------|
| 总CapEx × 85% | 假设 85% 是维持现有业务，15% 是扩张新业务 |
| 100% 折旧 | 每年设备损耗多少就得花多少来修 |
| 历史CapEx/营收比 × 当期营收 | 按历史比例推算 |

**算出所有者盈余后**，用它做一个简单的 DCF（5年增长 + 终端价值），折现率用 15%（巴菲特的高标准），最后再打 25% 的折扣作为安全边际。

**安全边际是什么？** 就是"我觉得值 100 块，但我只愿意花 75 块买"。多出来的 25% 就是万一我算错了也不亏。

### 5.3 方法三：EV/EBITDA 相对估值（权重 20%）

**思路不同**：不看公司"绝对值多少"，而是看"跟历史比，现在贵不贵"。

```
当前 EV/EBITDA = 公司总价值 / 息税折旧前利润

如果当前倍数 < 历史中位数 → 比历史平均便宜 → 可能被低估
如果当前倍数 > 历史中位数 → 比历史平均贵了 → 可能被高估
```

**举个例子**：苹果公司过去 10 年 EV/EBITDA 倍数的中位数是 18 倍。现在只有 15 倍。那说明现在比平时便宜。

算出隐含价值 = 历史中位数倍数 × 当前 EBITDA - 净债务

### 5.4 方法四：残余收益模型（权重 10%）

**思路**：一家公司值不值钱，看它能不能赚到"比股东期望的更多"。

```
账面价值 = 公司按净资产算值多少
残余收益 = 实际利润 - 股东期望的利润（= 权益成本 × 账面价值）

如果残余收益 > 0 → 公司在创造额外价值
如果残余收益 < 0 → 公司在毁损价值

公司价值 = 账面价值 + 未来所有残余收益的折现值
最后再打 8 折（20% 安全边际）
```

### 5.5 四种方法怎么合成？

每种方法都算出一个"合理价格"，然后和当前市值比较：

```
差距 = (合理价格 - 当前市值) / 当前市值

如果差距 > 15% → 被低估了 15% 以上 → 看涨（该买）
如果差距 < -15% → 被高估了 15% 以上 → 看跌（该卖）
其他            → 差不多合理 → 中立
```

然后四个方法加权：

```
综合差距 = DCF差距 × 35% + Owner Earnings差距 × 35% + EV/EBITDA差距 × 20% + 残余收益差距 × 10%
```

DCF 和 Owner Earnings 权重最大（各 35%），因为它们是"绝对估值"——直接算公司值多少钱，更严谨。EV/EBITDA 是"相对估值"——跟历史比，有参考价值但不绝对。残余收益模型比较学术，权重最低。

## 六、情绪分析师 — 看新闻和内部人动向的人

**纯数学，不用 AI。**

### 6.1 两个信息源

#### 信息源一：内部人交易（权重 30%）

**什么是内部人？** 公司的高管、董事、大股东。他们比任何人都了解公司真实情况。

- 内部人在**买入**自己公司的股票 → 他们觉得公司前景好 → 看涨信号
- 内部人在**卖出** → 他们觉得前景不好，或者股价到顶了 → 看跌信号

#### 信息源二：新闻情绪（权重 70%）

API 已经给每条新闻打了情绪标签：positive（正面）/ negative（负面）/ neutral（中性）。

- 正面新闻多 → 看涨
- 负面新闻多 → 看跌

### 6.2 为什么新闻权重 70%，内部人只有 30%？

因为新闻数量多（100条）且及时（每天都更新），内部人交易相对少（可能一个月才几笔）。

### 6.3 合成方法

```
加权看涨 = 内部人买入次数 × 0.3 + 正面新闻数量 × 0.7
加权看跌 = 内部人卖出次数 × 0.3 + 负面新闻数量 × 0.7

看涨 > 看跌 → 看涨
看跌 > 看涨 → 看跌
相等     → 中立
```

## 七、12 个"投资大师"AI

### 7.1 它们在干什么？

**一句话：让 AI 假装自己是巴菲特/林奇/...，然后根据财务数据给出判断。**

### 7.2 以巴菲特 AI 为例

巴菲特 AI 的工作分两步：

**第一步：用代码做 7 项分析**（和前面基本面分析师类似但更深入）

| 分析 | 在看什么 |
|------|---------|
| 基本面 | ROE、负债率、利润率、流动比率 |
| 一致性 | 过去几年盈利是不是一直在增长 |
| 护城河 | 公司有没有"别人抢不走"的优势（品牌、技术、规模） |
| 定价权 | 公司能不能涨价而不流失客户（看毛利率趋势） |
| 账面价值增长 | 每股净资产是不是一直在增加 |
| 管理层质量 | 管理层在回购股票（好）还是在稀释股权（坏） |
| 内在价值 | 用三阶段 DCF 算公司值多少钱 |

**第二步：把所有分析结果喂给 AI**

```
AI 收到的信息：
"你是巴菲特。苹果公司的数据如下：
  - ROE 30%，利润率 25%
  - 护城河评分 4/5
  - 内在价值 3万亿，当前市值 2.8万亿
  - 安全边际 7%
  请判断：看涨/看跌/中立？置信度多少？为什么？"

AI 回复：
"看涨，置信度 80%。苹果有极强护城河和优秀管理层，
 安全边际为正，在我的能力圈范围内。"
```

### 7.3 12 个大师有什么区别？

**代码部分几乎一样**（都拉同样的财务数据，做类似的计算）。区别只在给 AI 的 **system prompt**（角色设定）不同：

| 大师 | AI 被要求关注的重点 |
|------|-----------------|
| 巴菲特 | 护城河 + 安全边际 + 管理层 |
| 芒格 | 好生意 + 理性 + 长期 |
| 林奇 | "买你懂的" + PEG + 十倍股 |
| 阿克曼 | 激进主义 + 集中投资 |
| 凯瑟琳·伍德 | 颠覆性创新 + 高增长 |
| 布瑞 | 深度逆向 + 做空泡沫 |
| ... | ... |

**问题**：这 12 个 AI 其实是同一个模型在回答，只是"装"成了不同人。它们的回答很可能是类似的——都看到了同样的财务数据，用同样的模型推理，输出相似的结论。所以这 12 个"不同视角"没那么独立。

## 八、风控经理 — 管风险的人

**纯数学，不用 AI。这是整个项目设计最好的部分。**

风控经理要回答的问题是："每只股票最多能买多少？"

它从两个角度限制：

### 8.1 角度一：波动率限制（这只股票本身稳不稳定？）

**波动率** = 股价上蹿下跳的程度。

- 苹果股价每天只波动 ±1% → 波动率低 → 可以多买
- 某小盘股每天波动 ±5% → 波动率高 → 少买点

**具体规则**：

| 年化波动率 | 股票类型举例 | 最多能占总资产的比例 |
|-----------|------------|-----------------|
| < 15%（稳如老狗） | 大型蓝筹股、公用事业 | 25%（可以多买） |
| 15-30%（正常波动） | 大部分科技公司 | 12.5-20% |
| 30-50%（上蹿下跳） | 小盘成长股、加密货币 | 5-15% |
| > 50%（坐过山车） | 高投机性资产 | 最多 10% |

**举例**：你有 10 万美元。

- 苹果波动率 20% → 最多买 10万 × 17.5% = 17500 美元
- 某小盘股波动率 45% → 最多买 10万 × 10% = 10000 美元

### 8.2 角度二：相关性限制（这些股票会不会同涨同跌？）

**问题**：如果你买了苹果和微软，而这两只股票涨跌几乎同步（相关性 0.9），那你等于"把鸡蛋放在同一个篮子里"。

**相关性** 用 -1 到 +1 的数字表示：

- +1：完全同步（同涨同跌）
- 0：完全无关
- -1：完全相反（你涨我跌）

**规则**：

| 和现有持仓的平均相关性 | 怎么调整 |
|---------------------|---------|
| ≥ 0.8（几乎同步） | 大幅减仓（×0.70），太危险了 |
| 0.6-0.8（比较同步） | 适度减仓（×0.85） |
| 0.4-0.6（中等） | 不调整（×1.00） |
| 0.2-0.4（不太相关） | 微微加仓（×1.05），分散好 |
| < 0.2（完全不相关） | 鼓励多买（×1.10），分散风险 |

**举例**：你已经持有苹果和微软（都是科技股，相关性 0.85）。现在想买谷歌（也是科技股）。

- 谷歌和苹果/微软相关性 0.85 → 乘数 ×0.70
- 原本可以买 20000 → 实际只能买 20000 × 0.70 = 14000

风控在说："你已经有两个科技股了，再加一个太集中了，少买点。"

### 8.3 两个角度怎么叠加？

**乘法叠加**：

```
最终限额 = 总资产 × 基准比例(20%) × 波动率乘数 × 相关性乘数

然后：
可用额度 = 最终限额 - 你已经持有的市值
实际能买 = min(可用额度, 剩余现金)
```

**完整例子**：

```
你有 10 万美元现金，没有任何持仓。
想买英伟达（NVDA）。

第一步：算波动率
  NVDA 年化波动率 = 40%（属于高波动）
  波动率乘数 = 0.55
  基准限额 = 10万 × 20% × 0.55 = 11000

第二步：算相关性（因为你没有持仓，所以跳过）
  相关性乘数 = 1.0

第三步：最终限额
  = 11000 × 1.0 = 11000
  可用额度 = 11000 - 0（没持仓） = 11000
  实际能买 = min(11000, 100000) = 11000

结论：NVDA 最多买 11000 美元。
```

## 九、组合经理 — 拍板做决定的人

**这是唯一用 AI 做最终决定的环节。**

### 9.1 它收到什么信息？

所有分析师的结论 + 风控的限制：

```
AAPL 的信号汇总：
  技术分析师：看涨，置信度 70
  基本面分析师：看涨，置信度 60
  估值师：看跌，置信度 55
  情绪分析师：中立，置信度 50
  巴菲特AI：看涨，置信度 80
  林奇AI：看涨，置信度 65
  风控：最多买 $5000，当前价格 $178

允许的操作：买（最多28股）、持有
```

### 9.2 聪明的地方：不让 AI 做数学

风控已经用代码算好了"最多能买 28 股"。AI 只需要从"买"和"持有"中选一个，再选买多少股（不超过 28）。

**为什么不让 AI 算？** 因为 AI 做数学经常出错。让它算"我有 5000 块，股价 178，能买几股"，它可能算出 30 股（超过了预算）。用代码先算好上限，AI 只做选择，不做计算。

### 9.3 AI 怎么回答？

AI 被强制要求输出固定格式：

```json
{
  "AAPL": {
    "action": "buy",
    "quantity": 25,
    "confidence": 75,
    "reasoning": "多数分析师看涨，护城河强，价格合理"
  },
  "MSFT": {
    "action": "hold",
    "quantity": 0,
    "confidence": 60,
    "reasoning": "信号混合，等待更明确方向"
  }
}
```

用 Pydantic 库强制 AI 按这个格式输出，如果格式不对就重试（最多3次），还是不行就用默认值（全部持有）。

## 十、LLM 调用层 — 怎么调用 AI 的

### 10.1 统一封装

所有需要 AI 的地方都调同一个函数 `call_llm()`：

```
call_llm(问题, 期望的回答格式, agent名字, 状态) → 结构化的回答
```

它做了几件重要的事：

1. **支持多种 AI**：OpenAI、DeepSeek、Anthropic、Ollama（本地）等 12 种
2. **结构化输出**：用 Pydantic 模型告诉 AI "你必须按这个格式回答"
3. **重试机制**：AI 回答失败时重试最多 3 次
4. **兜底机制**：3 次都失败，返回一个安全的默认值（比如"中立，置信度 50"）

### 10.2 为什么需要兜底？

AI 不保证 100% 正常工作。可能是网络断了、API 额度用完了、返回格式不对。有了兜底机制，即使 AI 完全挂了，系统也不会崩溃——它只会说"中立"（不买不卖），这是最安全的做法。

## 十一、回测引擎 — 用历史数据验证策略

### 11.1 什么是回测？

**类比**：你发明了一套"看云识天气"的方法。在真正用它预测明天天气之前，你先拿过去 3 年的天气数据，假装你每天都用这个方法预测，看看准确率有多高。这就是回测。

### 11.2 回测怎么跑？

```
从起始日到结束日，每个工作日做一次：

1. 看当前各股票价格
2. 调用整个 Agent 系统做分析决策
3. 按决策"执行"交易（模拟的）
4. 记录今天组合的总价值
5. 计算绩效指标

跑完后，你能看到：
- 一开始 10 万，最后变成多少
- 最大的一次亏损是多少（最大回撤）
- 夏普比率（风险调整后的收益）
```

### 11.3 三个关键绩效指标

#### 夏普比率（Sharpe Ratio）

**衡量：每承受 1 单位风险，能赚多少超额收益。**

```
夏普比率 = (年化收益率 - 无风险利率) / 年化波动率
```

- > 1.0：好策略
- > 2.0：非常好的策略
- < 0：还不如存银行

**大白话**：赚得多但不稳定（大起大落），不如赚得少但很稳定。

#### 索提诺比率（Sortino Ratio）

**和夏普类似，但只惩罚下跌的波动。**

上涨的波动（暴涨）是好事，不应该被惩罚。索提诺只看下跌的标准差。

- 索提诺 > 夏普 → 上涨多、下跌少 → 更好

#### 最大回撤（Max Drawdown）

**从最高点到最低点，最多亏了多少百分比。**

```
比如你的组合价值变化：
10万 → 12万 → 9万 → 11万 → 13万

最高点是 12万，之后跌到 9万
最大回撤 = (9万 - 12万) / 12万 = -25%
```

- 最大回撤 < 10%：非常稳健
- 最大回撤 > 30%：相当刺激
- 最大回撤 > 50%：腰斩了

|------|------|--------|
| DCF 现金流折现 | 35% | 最严谨的"绝对估值"方法 |
| Owner Earnings | 35% | 巴菲特偏好的方法，跟 DCF 互补 |
| EV/EBITDA | 20% | 跟历史比，反映市场定价 |
| 残余收益 | 10% | 学术方法，辅助 |

### 情绪分析师内部的权重

| 来源 | 权重 | 为什么 |
|------|------|--------|
| 新闻情绪 | 70% | 数据量大、更新及时 |
| 内部人交易 | 30% | 信号质量高但数量少 |

### Agent 之间没有权重！

**重要**：不同 Agent（技术面 vs 基本面 vs 巴菲特AI vs ...）之间没有预设的权重。所有信号平等地交给组合经理 AI，由 AI 自己判断该听谁的。

这是一个缺陷——更好的做法是根据历史准确率给不同 Agent 分配不同权重。

## 十四、学到了什么？什么不该学？

### 该学的

1. **用 LangGraph 编排多个 AI Agent**：声明式定义流程，自动并行和同步
2. **技术指标计算**：EMA、布林带、RSI、ADX、Hurst 指数的纯 Python 实现
3. **风控设计**：波动率控制单票风险 + 相关性控制集中度风险
4. **不让 AI 做数学**：代码先算好约束，AI 只做选择
5. **估值多模型加权**：不同方法互补验证
6. **兜底机制**：AI 挂了也不会崩，返回安全的默认值

### 不该学的

1. **12 个大师 AI 是换皮**：同样的代码 + 不同的 prompt，产出高度相似
2. **AI 做最终交易决策不靠谱**：不可解释、不可复现
3. **没有历史准确率加权**：所有 Agent 平等，不区分谁更准
4. **回测结果可能是过拟合**：在已知数据上调参数，不代表未来有效

### 一句话总结

> **这个项目最有价值的部分是纯数学计算的部分（技术指标、基本面打分、估值模型、风控）。用 AI 做的部分（大师角色扮演、最终决策）更像是一个有趣的演示，但不适合真的用来做投资决策。**
