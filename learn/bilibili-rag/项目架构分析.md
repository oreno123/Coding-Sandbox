# Bilibili RAG 项目架构深度分析

> 面向快速掌握项目的新手友好型技术分析文档

---

## 1. 项目整体定位

### 1.1 项目类型

**全栈 Web 应用**（前后端分离）

- **后端**：独立的 FastAPI REST API 服务，提供 HTTP 接口
- **前端**：Next.js 单页应用（SPA），通过 `fetch` 调用后端 API
- **数据**：本地 SQLite + ChromaDB，无独立部署的数据库服务

### 1.2 核心功能和业务场景

| 功能模块 | 业务场景 | 用户价值 |
|---------|---------|---------|
| B 站扫码登录 | 获取用户收藏夹权限 | 安全、免密码 |
| 收藏夹读取 | 拉取用户收藏的视频列表 | 数据来源 |
| 音频转写（ASR） | 将视频语音转为文字 | 内容可检索 |
| 向量入库 | 文本分块 → Embedding → 存入 ChromaDB | 语义检索基础 |
| RAG 对话 | 检索 + LLM 生成回答 | 基于收藏内容的智能问答 |
| 收藏夹整理 | 预览/批量移动/清理失效 | 辅助管理收藏夹 |

**核心业务流程**：`登录 → 选收藏夹 → 入库（拉取+ASR+向量化）→ 对话问答`

### 1.3 目标受众和使用场景

- **个人学习者**：收藏了大量访谈/课程/演讲，想快速检索和复习
- **知识管理**：把 B 站收藏变成可对话的知识库
- **技术研究**：学习 RAG、向量检索、LLM 应用的实际案例

### 3.1 核心目录结构

```
bilibili-rag/
├── app/                    # 后端主目录
│   ├── main.py             # 应用入口、路由注册、中间件
│   ├── config.py           # 配置（Settings）
│   ├── database.py         # 数据库引擎、会话工厂
│   ├── models.py           # 数据模型（SQLAlchemy + Pydantic）
│   ├── routers/            # API 路由

│   │   ├── favorites.py    # 收藏夹 / 视频列表 / 整理
│   │   ├── knowledge.py   # 知识库构建 / 同步 / 入库
│   │   └── chat.py        # 问答 / 流式输出 / 搜索
│   └── services/          # 业务逻辑层
│       ├── bilibili.py     # B 站 API 封装

│   ├── app/                # Next.js App Router
│   │   ├── page.tsx       # 首页
│   │   └── layout.tsx     # 布局
│   ├── components/        # 组件
│   │   ├── LoginModal.tsx  # 登录弹窗
│   │   ├── SourcesPanel.tsx  # 收藏夹面板
│   │   ├── ChatPanel.tsx  # 对话面板
│   │   ├── OrganizePreviewModal.tsx
│   │   └── DemoFlowModal.tsx
│   └── lib/
│       └── api.ts         # API 客户端封装
├── data/                   # 运行时数据（SQLite、ChromaDB）
├── test/                   # 测试脚本（需移动到根目录运行）
├── .env.example            # 环境变量模板
└── requirements.txt
```

### 3.2 关键文件作用

| 文件 | 用途 |
|------|------|
| `app/main.py` | 创建 FastAPI 应用、注册路由、CORS、lifespan 启动/关闭 |
| `app/config.py` | `Settings` 类，从 `.env` 读取 API Key、模型、数据库路径等 |
| `app/database.py` | 创建异步引擎、`get_db` 依赖、`async_session_factory` |
| `app/models.py` | 表结构（VideoCache、UserSession、FavoriteFolder 等）和 Pydantic 请求/响应模型 |
| `app/routers/*.py` | 各模块的 HTTP 接口定义 |
| `app/services/content_fetcher.py` | **内容获取策略**：ASR → 基本信息兜底 |
| `app/services/rag.py` | **RAG 核心**：文本分块、向量化、检索、LLM 问答 |
| `frontend/lib/api.ts` | 封装所有后端 API 的 `fetch` 调用 |

### 3.3 模块划分逻辑

采用**按功能域划分**的模块化设计：

- **auth**：认证、登录、会话
- **favorites**：收藏夹、视频、整理
- **knowledge**：知识库构建、同步、入库状态
- **chat**：问答、流式输出、搜索

每个模块内部：`router` 负责 HTTP，`service` 负责业务逻辑。

## 4. 核心代码逻辑

### 4.1 项目入口流程

```
用户访问 http://localhost:3000
    ↓
Next.js 渲染 page.tsx
    ↓
未登录 → 显示 hero 区 + 扫码登录按钮
已登录 → 显示 workspace（SourcesPanel + ChatPanel）
    ↓
用户点击「入库」→ SourcesPanel 调用 knowledgeApi.build
    ↓
后端 knowledge.build 创建后台任务
    ↓
_sync_folder 遍历收藏夹视频
    ↓
ContentFetcher.fetch_content（ASR / 基本信息）
    ↓
RAGService.add_video_content（分块 + 向量化 + 入库）
    ↓
用户提问 → ChatPanel 调用 /chat/ask/stream
    ↓
chat._prepare_messages（路由决策）
    ↓
RAGService.search 或 数据库查询
    ↓
LLM 生成回答，流式返回
```

### 4.2 核心业务逻辑

#### （1）内容获取策略（ContentFetcher）

```
fetch_content(bvid, cid, title)
    ├── 获取视频基本信息（补全 cid/title）
    ├── 优先 ASR：audio_url 可达 → Transcription(url)
    │   不可达 → 本地下载 → ffmpeg 转码 → Recognition(本地文件)
    ├── ASR 成功 → 返回 VideoContent(source=ASR)
    └── 失败 → 兜底：标题 + 简介 → VideoContent(source=BASIC_INFO)
```

**代码位置**：`app/services/content_fetcher.py` 第 33–98 行

#### （2）RAG 流程（RAGService）

```
add_video_content(video)
    ├── 拼接标题 + 正文 + 提纲
    ├── RecursiveCharacterTextSplitter 分块（1000 字符，200 重叠）
    ├── 每块 → Document(page_content, metadata)
    └── vectorstore.add_documents

search(query, k, bvids)
    └── vectorstore.similarity_search(query, k, filter={"bvid": {"$in": bvids}})
```

**代码位置**：`app/services/rag.py` 第 120–196 行、第 233–256 行

#### （3）对话路由策略（chat.py）

```
_prepare_messages
    ├── LLM 路由：direct / db_list / db_content / vector
    ├── 规则兜底：列表/总结/向量检索
    ├── direct → 通用回答（不查库）
    ├── db_list → 用标题/简介回答列表类问题
    ├── db_content → 用内容回答总结类问题
    └── vector → 向量检索 → RAG 回答
```

**代码位置**：`app/routers/chat.py` 第 381–409 行

### 4.3 关键技术点示例

#### 示例 1：流式问答（SSE 风格）


#### 示例 2：收藏夹同步逻辑


## 5. 学习建议

### 5.1 入门学习路径（按优先级）

1. **FastAPI 基础**：路由、依赖注入、Pydantic 模型
2. **SQLAlchemy 异步**：`create_async_engine`、`AsyncSession`、`get_db`
3. **LangChain 概念**：Document、TextSplitter、Embeddings、VectorStore
4. **RAG 流程**：检索 → 上下文构建 → 提示词 → LLM 生成
5. **Next.js App Router**：`page.tsx`、`layout.tsx`、`"use client"`
6. **React 状态**：`useState`、`useEffect`、`useCallback`

### 5.2 核心难点和重点

| 难点 | 说明 | 建议 |
|------|------|------|
| **内容获取策略** | ASR 优先、URL 可达性、本地兜底 | 重点看 `content_fetcher.py` 的 `_try_asr` 和 `_try_asr_with_local_audio` |
| **对话路由** | LLM 路由 + 规则兜底 | 理解 `_route_with_llm` 和 `_route_with_rules` 的配合 |
| **跨 Session 关联** | 同一用户多 session 的收藏夹去重 | 看 `_get_folder_ids_for_session` 的 mid 查询 |
| **向量过滤** | 按 `bvids` 过滤检索范围 | 理解 ChromaDB 的 `filter` 参数 |

### 5.3 可优化点

| 方向 | 建议 |
|------|------|
| **架构** | 会话存储改为 Redis，支持多实例部署 |
| **性能** | 批量入库时控制并发，避免 ASR 限流 |
| **可维护性** | 将 `chat.py` 中的路由逻辑抽到独立 service |
| **可扩展性** | 抽象 LLM/Embedding 接口，支持多模型切换 |
| **功能** | 实现 TodoList：对话历史、分 P 视频、多模型适配 |
