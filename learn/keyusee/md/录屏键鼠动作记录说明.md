# 录屏键鼠动作记录说明

## 1. 记录链路（从采集到落盘）

1. 渲染进程开始录屏时调用 `window.api.startInputTracking()`，主进程收到 `start-input-tracking` 后启动 `uIOhook.start()`。
2. 主进程通过 `uIOhook.on('input', ...)` 监听全局输入事件，并通过 `mainWindow.webContents.send('input-event', e)` 转发给渲染进程。
3. 渲染进程通过 `window.api.onInputEvent(...)` 订阅事件，将每条事件追加到内存数组，并额外写入 `timestamp: Date.now()`。
4. 停止录屏时，渲染进程先取消监听，再调用 `window.api.stopInputTracking()`，主进程执行 `uIOhook.stop()`。
5. 保存时将事件数组 `JSON.stringify(inputEventsRef.current, null, 2)` 序列化，随录屏文件一起写入目录，文件名固定为 `键鼠操作.json`。

---

## 2. 记录了哪些动作类型

底层来源是 `uiohook-napi` 的 `input` 事件，项目当前会记录以下类别：

- 键盘按下：`type = 4`（`EVENT_KEY_PRESSED`）
- 键盘抬起：`type = 5`（`EVENT_KEY_RELEASED`）
- 鼠标点击：`type = 6`（`EVENT_MOUSE_CLICKED`）
- 鼠标按下：`type = 7`（`EVENT_MOUSE_PRESSED`）
- 鼠标抬起：`type = 8`（`EVENT_MOUSE_RELEASED`）
- 鼠标移动：`type = 9`（`EVENT_MOUSE_MOVED`）
- 鼠标滚轮：`type = 11`（`EVENT_MOUSE_WHEEL`）

补充说明：

- 底层拖拽事件 `EVENT_MOUSE_DRAGGED (10)` 在 `uiohook-napi` 绑定层会被改写为 `EVENT_MOUSE_MOVED (9)`，因此最终日志中只看到 `type=9`。
- 键盘字符输入事件 `EVENT_KEY_TYPED (3)` 不在当前转发范围内，因此不会出现在日志中。

---

## 3. 记录规则

### 3.1 统一规则

- 每个输入事件记录为 JSON 数组中的一个对象（按发生顺序 append）。
- 事件对象保留 `uiohook-napi` 原始字段，同时增加业务时间戳 `timestamp`。
- 不做去重、节流或采样，事件会完整写入（高频移动会产生大量记录）。
- 只在“录制中”采集；停止录制后不再追加事件。

### 3.2 时间字段规则

- `time`：来自 `uiohook` 原始事件时间（库输出字段）。
- `timestamp`：渲染进程收到事件时写入的 `Date.now()`（毫秒时间戳）。

建议后续消费日志时优先使用 `timestamp` 作为统一业务时间轴；若做底层对齐，可结合 `time` 做交叉校验。

### 3.3 按类型字段规则

- 键盘事件（`type=4/5`）包含：
  - `keycode`
  - 修饰键状态：`altKey`、`ctrlKey`、`metaKey`、`shiftKey`
  - 时间字段：`time`、`timestamp`
- 鼠标移动/按压/释放/点击（`type=6/7/8/9`）包含：
  - 坐标：`x`、`y`
  - 按键信息：`button`、`clicks`
  - 修饰键状态与时间字段
- 滚轮事件（`type=11`）包含：
  - 坐标：`x`、`y`
  - 滚轮参数：`clicks`、`amount`、`direction`、`rotation`
  - 修饰键状态与时间字段

## 5. 当前实现特性与边界

- 当前日志是“原始输入事件流”，没有转换为“用户语义动作”（如“双击选中”“拖动选区”）。
- 鼠标拖拽与普通移动在日志中统一为 `type=9`，若要区分拖拽，需要后续结合按键状态与轨迹自行推断。
- 高频输入会导致日志较大，属于预期行为。
